# Security Baseline

Security starts before application code exists.

## Repository Rules

- Do not commit secrets.
- Do not commit real user data.
- Do not commit private credentials.
- Treat authentication, authorization and tenant isolation as architecture topics.
- Document sensitive data flows before implementation.

## Future Security Controls

- Secret scanning.
- Dependency vulnerability scanning.
- Authorization test coverage.
- Audit logging policy.
- Data retention policy.
- Incident response process.
