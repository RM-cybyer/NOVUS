# Naming Standards

Use consistent names to keep the repository navigable.

## Current Standards

| Item | Standard |
| --- | --- |
| Components | PascalCase |
| Functions | camelCase |
| Folders | kebab-case |
| Interfaces | PascalCase |
| Types | PascalCase |
| Constants | UPPER_SNAKE_CASE |
| Files | kebab-case |

## Documentation Files

Documentation files should use kebab-case:

```text
repository-governance.md
security-baseline.md
```

## Exceptions

Framework-required filenames may use framework conventions. Document framework conventions after a runtime is selected.
