# Documentation Standards

Documentation must stay close to repository reality.

## Required Documentation

- New folders require README files.
- Architecture changes require documentation updates.
- Boundary-changing decisions require ADRs.
- Major features require blueprints.
- Security-sensitive changes require security notes.

## Writing Rules

- Write for future maintainers.
- State what exists now versus what is planned.
- Include non-goals when ambiguity is likely.
- Use templates for repeated documentation types.
- Avoid promising behavior that has not been implemented.
