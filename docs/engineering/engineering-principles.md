# Engineering Principles

NOVUS OS engineering work should optimize for long-term repository health.

## Principles

- Clarity before cleverness.
- Boundaries before velocity.
- Documentation with decisions.
- Security before sensitive data.
- Tests before scale.
- Simple abstractions until complexity is proven.
- Local consistency over imported preference.

## Non-principles

- Do not optimize prematurely.
- Do not add tooling because it is fashionable.
- Do not implement product behavior during foundation work.
