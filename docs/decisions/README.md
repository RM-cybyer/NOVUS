# Architecture Decision Records

This folder contains architecture decision records, also called ADRs.

## Responsibility

- Record decisions that affect repository structure, module boundaries, tooling, runtime, security, data model or long-term maintainability.
- Preserve context for future engineers and AI agents.

## Naming

Use numbered kebab-case filenames:

```text
0001-repository-operating-system.md
0007-official-runtime-framework.md
```

## Current Runtime Decision

ADR 0007 defines the official NOVUS runtime framework, technology stack, provider-neutral AI layer, package manager, testing strategy and deployment target.
