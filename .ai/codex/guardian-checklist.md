# Guardian Checklist

Use this checklist before finalizing repository work.

## Before Changes

- Read `.ai/shared/`.
- Read `.ai/codex/`.
- Inspect `git status`.
- Identify whether the task is repository health, product implementation or strategy.
- Confirm whether an ADR or blueprint is required.

## During Changes

- Keep edits scoped.
- Avoid application code unless explicitly approved by a blueprint.
- Add README files for new folders.
- Update architecture docs when structure changes.
- Prefer templates over ad hoc process text when the pattern will repeat.

## Before Final Response

- Verify required paths exist.
- Check for non-ASCII characters unless intentionally needed.
- Confirm no secrets were introduced.
- Summarize changes, decisions and recommended next steps.
