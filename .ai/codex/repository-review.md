# Repository Review Procedure

Use this procedure for Repository Guardian reviews.

## Review Areas

- Folder ownership is clear.
- Documentation is current.
- Templates support future work.
- Role boundaries are preserved.
- No product logic lives in routing, documentation or configuration placeholders.
- Security guidance exists before sensitive implementation begins.
- Testing and release strategy have a documented path.

## Evidence to Gather

- `git status --short`
- File tree for changed areas
- Relevant READMEs
- Relevant ADRs or blueprints
- Workflow configuration

## Output Format

Lead with risks or blockers. Then summarize improvements, verification and recommended follow-up.
