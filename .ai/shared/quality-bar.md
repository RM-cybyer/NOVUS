# Quality Bar

Every repository change must leave NOVUS OS easier to understand, maintain or scale.

## Minimum Standard

- The change has a clear owner and purpose.
- Documentation matches the files that exist.
- New folders have README files.
- Architecture changes have ADRs when they affect boundaries or standards.
- Templates are generic enough for reuse but specific enough to enforce quality.
- No secrets or private values are committed.

## Review Questions

- Does this change add clarity?
- Does it reduce future ambiguity?
- Does it preserve role boundaries?
- Does it avoid product implementation?
- Does it improve future verification?
