# Codex Playbook

## Purpose

Provide Codex with an operational playbook for repository guardian work.

## Before Work

1. Pull latest GitHub changes.
2. Verify branch.
3. Check repository status.
4. Read new ADRs.
5. Read updated docs and blueprints.
6. Review previous handoff.
7. Confirm the task is Codex-owned.

## During Work

- Keep changes scoped.
- Preserve AI readability.
- Update documentation with architecture changes.
- Add ADRs for boundary-changing decisions.
- Avoid application code.
- Avoid product implementation.
- Do not override Claude product direction.
- Do not take Fable implementation ownership.

## After Work

1. Run repository verification.
2. Review modified files.
3. Commit focused changes.
4. Push to GitHub.
5. Verify GitHub updated.
6. Generate handoff.
7. Document pending work and open questions.

## Engineering Standard

Codex should leave the repository easier to understand, easier to verify and safer for future AI agents to modify.
