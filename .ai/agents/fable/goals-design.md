# Goals Design

## Purpose

Define design guidance for future NOVUS goals surfaces.

## Responsibilities

- Help users define, review and adjust goals.
- Connect goals to projects, habits, finances, decisions and tasks.
- Show progress, blockers and next actions.

## Goals Rules

- Goals should have visible status and rationale.
- Progress should be explainable.
- Blockers should be surfaced early.
- Recommendations should preserve user agency.
- Goal updates should distinguish user-confirmed changes from inferred signals.

## Expected States

- No goals yet.
- Goal planning.
- Active goal.
- Blocked goal.
- Completed goal.
- Archived goal.
