# Fable UI Philosophy

## Purpose

Define the user interface philosophy for NOVUS OS before product UI is implemented.

## Core Principle

NOVUS is an AI Life, Business and Financial Operating System. The UI should help users make better decisions, not distract them with decoration or behave like a generic chatbot.

## Responsibilities

- Keep decision-making visible.
- Make life, business and financial context easy to scan.
- Support trust through clarity, provenance and trade-offs.
- Design workflows that help users act, review and improve over time.

## UI Rules

- Prioritize clarity over spectacle.
- Use dense but calm layouts for operational surfaces.
- Make trade-offs, risks and assumptions visible.
- Keep AI recommendations explainable.
- Do not hide important state inside conversational text.
- Avoid purely decorative UI that does not improve decisions.

## What Belongs Here

- UI philosophy.
- Design principles.
- Product experience rules.
- Interface decision criteria.

## What Never Belongs Here

- Component implementation.
- CSS or framework code.
- Product roadmap decisions.
- Brand assets.
