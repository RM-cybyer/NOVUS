# Transactions Design

## Purpose

Define design guidance for future financial transaction surfaces.

## Responsibilities

- Make financial data scannable and trustworthy.
- Support categorization, review, anomaly detection and decision-making.
- Show uncertainty and source quality.

## Transaction Rules

- Treat financial data as sensitive.
- Show source and sync status where relevant.
- Make categories reviewable.
- Do not overstate financial conclusions.
- Surface anomalies without panic.
- Keep auditability in mind.

## Expected States

- Loading transactions.
- Empty state.
- Partial sync.
- Review required.
- Categorized.
- Anomaly detected.
- Error or disconnected source.

## AI Ownership

- Owning AI agent: Fable.
- Collaborating AI agents: Codex for security/runtime boundaries, Claude for product strategy, QA for validation.
