# Fable Design Review

## Purpose

Define the design review process for user-facing NOVUS work.

## Responsibilities

- Validate that UI supports decision-making.
- Check alignment with design philosophy.
- Check accessibility and responsive behavior.
- Confirm motion is purposeful.
- Confirm implementation follows repository architecture.

## Review Checklist

- Does the UI help users make a better decision?
- Are risks, trade-offs and uncertainty visible where needed?
- Is the layout scannable?
- Are interaction states clear?
- Does text fit on mobile and desktop?
- Is motion purposeful and accessible?
- Are empty, loading, error and success states defined?
- Does the implementation respect folder ownership?
- Does the handoff document QA needs?

## Required Outputs

- Review summary.
- Issues found.
- Follow-up tasks.
- QA notes.
- Codex or Claude escalation if needed.

## AI Ownership

- Owning AI agent: Fable.
- Collaborating AI agents: QA, Codex, Claude.
