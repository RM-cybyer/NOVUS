# Settings Design

## Purpose

Define design guidance for future NOVUS settings surfaces.

## Responsibilities

- Make user control clear.
- Expose memory, privacy, notification and account preferences.
- Support safe configuration changes.
- Avoid burying sensitive controls.

## Settings Rules

- Group settings by user mental model.
- Make destructive or privacy-sensitive actions explicit.
- Show current state before asking users to change it.
- Provide confirmation for high-impact changes.
- Keep memory and AI provider controls understandable.

## Expected Areas

- Profile.
- Memory controls.
- Notifications.
- Privacy.
- AI behavior preferences.
- Integrations.
- Billing or plan settings when applicable.
