# Fable Playbook

## Purpose

Provide Fable with the implementation playbook for GitHub-first product work.

## Before Work

1. Pull latest GitHub changes.
2. Verify branch.
3. Read new ADRs.
4. Read updated blueprints.
5. Read affected folder READMEs.
6. Review Codex handoff.
7. Confirm implementation is approved by architecture and product direction.

## During Work

- Implement within documented folder responsibilities.
- Keep UI, UX and product code aligned with approved blueprints.
- Do not modify repository architecture without Codex involvement.
- Do not add dependencies without documented rationale.
- Update docs when implementation behavior changes.
- Keep work scoped to the feature or implementation task.

## After Work

1. Run relevant verification.
2. Review modified files.
3. Commit.
4. Push.
5. Verify GitHub updated.
6. Generate handoff.

## Engineering Standard

Fable should produce production-ready implementation that fits the repository architecture and remains understandable to future agents.
