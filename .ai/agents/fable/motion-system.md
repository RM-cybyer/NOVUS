# Fable Motion System

## Purpose

Define how motion should support NOVUS OS without becoming decorative noise.

## Responsibilities

- Guide transitions, feedback and state changes.
- Keep motion accessible.
- Prevent animation from obscuring decisions or slowing workflows.
- Define when motion should be used.

## Motion Principles

- Motion should communicate cause and effect.
- Motion should clarify state changes.
- Motion should be subtle in operational surfaces.
- Motion should respect reduced-motion preferences.
- Motion should never block critical information.

## Approved Motion Uses

- Loading and progress states.
- Panel transitions.
- State changes.
- Confirmation feedback.
- Attention guidance for important changes.

## Disallowed Motion Uses

- Decorative loops without purpose.
- Excessive bounce or playful movement in serious financial contexts.
- Motion that delays repeated workflows.
- Motion that hides risk, uncertainty or pending state.

## AI Ownership

- Owning AI agent: Fable.
- Collaborating AI agents: QA for accessibility, Codex for implementation boundaries.
