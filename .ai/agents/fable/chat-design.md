# Chat Design

## Purpose

Define design guidance for conversational surfaces in NOVUS.

## Core Rule

NOVUS is not a chatbot. Conversation is one interface into the operating system, not the whole product.

## Responsibilities

- Make context, decisions and recommendations visible beyond message bubbles.
- Support structured outputs.
- Preserve memory transparency.
- Keep tool execution and completed actions clear.

## Chat Rules

- Show when NOVUS is reasoning, retrieving context or executing tools.
- Distinguish recommendation, explanation and action.
- Never imply a tool action occurred unless runtime executed it.
- Provide structured follow-up options when appropriate.
- Keep critical financial, business or life trade-offs visible.

## AI Ownership

- Owning AI agent: Fable.
- Collaborating AI agents: Codex for runtime boundaries, Claude for product behavior, QA for validation.
