# Claude Handoff

## Purpose

Define the handoff format Claude must leave after strategy, roadmap or CTO review work.

## Required Handoff

```text
Agent: Claude
Branch or GitHub reference:
Latest GitHub commit or document:

What changed:
- 

Why it changed:
- 

Product decisions:
- 

Technical trade-offs:
- 

Affected files or docs:
- 

Breaking changes:
- 

New tasks for Codex:
- 

New tasks for Fable:
- 

Pending work:
- 

Future improvements:
- 

Open questions:
- 
```

## Handoff Rules

- Claude handoffs should create clear downstream tasks.
- Product direction must be explicit enough for Codex and Fable to act.
- If implementation should not start yet, state the blocker.
